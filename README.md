# Принцип работы:
Сервер выделил место под группу, каждый пользователь зарегистрировался и получил уникальный id.  
Тот кто создает группу добавил в выделенное место на сервере пользователей по id. Больше id не используется.  
На сервере хранится отсортированный словарь примерно такого вида:  
* {id_группы : {
  - id_сообщения1 : {
    - id_пользователя,  
    - id_сообщения на сообщение на которое идет ответ,  
    - тип сообщения(фото/текст),  
    - текст или фото},  
  - id_сообщения2 : {
    - тоже самое}}}

# Описание файлов программы и их функций
* main.py - запускает UI, пдгружает другие классы
* database.py - класс базы данных, желатльно только тех, групп, в которых пользователь состоит. Два метода: 
  - loadFiles - получить данные в формате .json и отправить на сервер
  - uploadFiles - файлы БД подгрузить в базу данных
* message.py - класс сообщения. При инициализации передается id_сообщения. Методы:
  - createMssage - получает id_пользователя, id_сообщения на которое идет ответ (или None), тип сообщения (текст или изображение), content (string, list[string]), возращает словарь формата как представлен выше
  - reWriteMessage - получает content (string, list[string]), возращает словарь формата как представлен выше
  - deleteMessage - удаляет сообщение, возращает пустой словарь

# Файловая структура
Главное окно приложения находится в main.py и может взаимодействовать с остальными окнами и базой данных. 
В отдельном файле создаётся класс Database или что-то типа того и база данных один раз подключается, в классе находятся методы для работы с базой данных.  
Не может взаимодействовать ни с чем.  
В разных файлах находятся заготовки frame (если на tkinter) для интерфейса различных окон, например окна с чатами, настройки, регистрации и т.д. не могут взаимодействовать ни с чем.  
Плюс вспомогательные классы по типу сообщений.  
И всё это на сиглтоне что бы классы не перемешивались.
