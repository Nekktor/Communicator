# Описание классов проекта
## Message - класс сообщения
 * Параметры:
   - type
   - content
   - replyIdMes
   - userId
   - timeChanges
 * Методы:
   - change; получает: content
   - delete
## User - класс пользователя
 * Параметры:
   - idUser
   - username
   - lastname (возможен None), (необязательная по началу функция)
   - birstday: date(year, month, day), (возможен None), (необязательная по началу функция)
   - timeCreating (необязательная по началу функция)
   - avatarUrl (необязательная по началу функция)
   - phone (возможен None), (необязательная по началу функция)
   - mail (возможен None), (необязательная по началу функция)
   - lastTimeOnline
 * Методы:
   - change
## Chat - класс одного чата
 * Параметры:
   - idChat
   - name
   - avatarUrl (необязательная по началу функция)
   - participants (id)
   - timeCreating (необязательная по началу функция)
   - messages (объекты класса Message)
 * Методы:
   - Создать
   - Удалить
   - Добавить сообщение (связь с Message)
   - Изменить (name или avatarUrl)
## Connection - класс подключения сервера
 * Функционал: поддерживает связь с сервером, занимается отправкой и получением данных
## SynqQueue - класс очереди операций, отправляемых на сервер
 * Функционал: получает операцию на вход, создает из них очередь по важности (отправка сообщения > отправка реакции)
## Notification - класс уведомления (необязательная по началу функция)
 * Параметры:
   - idNot
   - userId (получатель)
   - type
   - data (информация уведомления)
   - isRead
   - timeCreating

## Registration - класс регистрации в приложении
 * Параметры:
   - name
   - username
   - lastname (возможен None), (необязательная по началу функция)
   - birstday: date(year, month, day), (возможен None), (необязательная по началу функция)
   - phone (возможен None), (необязательная по началу функция)
   - mail (возможен None), (необязательная по началу функция)
   - timeCreating (необязательная по началу функция)
* Методы:
  - Проверка уникальности username
  - Подтверждение номера телефона (необязательная по началу функция)
  - Подтверждение почты (необязательная по началу функция)
  - Добавление данных в БД
